# 使用 ClientRouter 保留双层音频播放器

正式声景由三首古琴顺序循环的主层与四选一、默认关闭的环境层组成。播放器不自动播放，初始不设置媒体 `src`，不提供依赖 Range 响应的可拖动时间轴；只部署小于 25 MiB 的 MP3。

全站接受 Astro ClientRouter 的页面交换模型，并以 `transition:persist` 保留播放器节点。所有页面脚本在 `astro:page-load` 重新初始化，程序化内部导航使用 Astro `navigate()`。主题、曲目、环境选择和双层音量可恢复，但硬刷新与页面重新进入前台都保持暂停。任何素材必须先通过用户试听批准，候选音频不会进入正式播放器。
